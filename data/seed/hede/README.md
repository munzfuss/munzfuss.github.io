# Hede seed data ÔÇö Danish royal-mint coins from danskmoent.dk

Coin entries lifted from the parsed Hede 1971 catalogue cache
(`scripts/cache/hede/*.json`) shaped as Coin-schema records ready
to paste into `data/locations/denmark.yml` (or `schleswig_holstein.yml`
once a separate Schleswig-Holstein seed is generated).

**Not auto-loaded by the build** ÔÇö `scripts/build.py` walks
`data/locations/*.yml` only. These files live outside that prefix
on purpose: they are an intake staging area, not a published surface.

## Seed conventions

Mirrors the existing `seed_unsorted` bulk-ucoin block at the bottom
of `data/locations/denmark.yml`:

* **id**: `dk-hede-<volume><number>` ÔÇö e.g. `dk-hede-c5h120` is the
  1683 Speciedaler under Christian V, Hede 120.
* **fuss / phase**: `seed_unsorted` / `A` ÔÇö placeholder bucket. On
  promotion, reclassify each coin into the actual M├╝ntzfu├ƒ window
  (use the `Marken fin udbragt til` value from
  `scripts/cache/hede/<basename>.json` as the M├╝ntzfu├ƒ indicator ÔÇö
  9 dlr Ôëê 9-Fu├ƒ, 9.288 spdlr Ôëê 9┬╝-Fu├ƒ, 11.333 rd Ôëê 11Ôàô-Fu├ƒ, etc.).
* **All `*_verified` flags `false`** ÔÇö Hede-seed data needs a
  per-coin sanity check (specimen variance, sub-variant fineness,
  per-year edge cases) before any flag flips to `true`.
* **Source**: `type: literature`, URL pointing at the cached
  `danskmoent.dk/{chr,fr}/<basename>.htm` page.
* **`issuing_entity: danish_realm`** ÔÇö adjust to `gesamtstaat` on
  promotion when the coin is dated to the Danish-Norwegian Helstat
  period (1660ÔÇô1814) and was minted at Kopenhagen for joint
  circulation.

## Promotion path

For each entry you accept into `data/locations/denmark.yml`:

1. **Re-bucket fuss + phase.** Drop `seed_unsorted` / `A`; pick the
   M├╝ntzfu├ƒ and phase that match the coin's
   `Marken fin udbragt til` and year. Add a new phase to
   `denmark.yml` if the period isn't covered yet.
2. **Cross-check against Hede page.** Open the URL in `sources[0]`
   and confirm `nominal`, `weight_rough_g`, `fineness`, and
   `year_label`. The seed is verbatim from danskmoent.dk; the page
   may carry sub-variants the seed flattened.
3. **Add a `bruun_*` citation** when Bruun Part II (the 2025
   Stack's Bowers catalogue) lists a specimen ÔÇö `bruun_collection_id`
   primary, `bruun_part` / `bruun_lot_no` / `bruun_page` for
   look-up. The Bruun cache (`scripts/cache/bruun/`) is the source.
4. **Flip `*_verified: true`** field-by-field as each value is
   confirmed by an acceptable source per CLAUDE.md ┬º5 (coin
   inscription, museum catalogue, Hede + Bruun cross-attestation,
   Sieg, Numista entry with direct data, etc.).
5. **Run** `python scripts/build.py --validate-only` after each
   batch of integrations to catch chronology mismatches, missing
   phase definitions, or schema drift.

## Regenerate

```bash
python scripts/parse_hede.py --force          # if parser changed
python scripts/maintenance/build_hede_denmark_seed.py
```

The seed builder is idempotent ÔÇö overwrites `denmark.yml` from the
current state of the parsed cache. Re-run safe.

## Excluded from the Denmark seed

Filtered out at build time (would belong in a separate seed for the
relevant location):

* **Schleswig-Holstein mints**: Gl├╝ckstadt, Altona, Flensborg,
  Haderslev, Rethwitsch, Rendsborg ÔÇö destined for a future
  `schleswig_holstein.yml` seed.
* **Sk├Ñne / Norway**: Christiansstad, Altona+Kongsberg ÔÇö
  region-ambiguous, defer.
* **No mint extracted** ÔÇö pages where the H1 parser couldn't
  disambiguate the mint segment (typically multi-section catalogue
  pages or overview pages).
* **Multi-nominal pages with unparseable nominal-list** ÔÇö pages
  whose H1 carries a comma-list (e.g. ┬½1, 2, 3 og 4 Speciedaler┬╗)
  that the regex couldn't deterministically pair to per-sub-Hede
  denominations.
* **No specs block** ÔÇö very small Skilling / Hvid entries where
  danskmoent.dk doesn't publish Brutto / Finhed data.
