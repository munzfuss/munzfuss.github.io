## Insights / observations

**The ucoin rate-limit symptom WAS the slug-redirect, not URL
breakage.** The original 2026-05-11 diagnosis in
`scripts/maintenance/ucoin_fetch_composition.py` header was
«ucoin's slug routing changed and our `_url_index.json` is stale».
That was actually wrong. The cookie-clear test today proved it:
all 45 previously-status_404 tids re-fetched cleanly with the
SAME cached slugs after cookies cleared. The «wrong-tid canonical»
response was ucoin's anti-abuse mechanism, not a routing failure.
Mental model correction logged in `SOURCES.md §13.2`. Script
header still has the obsolete diagnosis but that's now flagged in
§13.

**Cookie-clear can make Cloudflare WORSE, not better.** Counter-
intuitive but observed: clearing cookies between sessions 3 and 4
removed our `cf_clearance` token (Cloudflare's «passed challenge»
proof). Session 4 launched with a fresh-fingerprint, suspicious-
profile request → straight 403 + JS challenge. If we had NOT
cleared cookies between session 3 and 4, the still-warm
`cf_clearance` MIGHT have carried through (untestable now). The
operational lesson: clear cookies for rate-limit recovery; do NOT
clear cookies if Cloudflare is the gate.

**Pattern B / one-Krause-KM-one-entry held up under multi-sub-
letter cases.** Case 9 was the most complex dedup so far (4 sub-
letters, two Krause splits, multi-source weight divergence). The
pattern handled it cleanly. Confidence that the remaining 37 cases
will follow the same shape — Hede sub-letters mostly differ on
mintmark within a coherent type — is now high.

**Hede's spec-card-for-the-base-type pattern is a real trap.** The
c4h47 case (silver spec card co-listing a Guldafslag in the
Zincksamlingen line) is unlikely to be unique. Any Hede page that
mentions «Guldafslag» / «Sølvafslag» / «cf.» variants in the
Zincksamlingen list should flag for cross-metal verification
before adopting the spec card values onto a different-metal entry.
TODO candidate: audit all curated entries whose `catalog.hede`
matches a sub-letter that the Hede page documents as off-strike.

**The five-file doc split immediately demonstrated value.** Within
the same session that the split was introduced: PLAYBOOKS PB-2
captured the ucoin harvest pacing rules into one executable place,
SOURCES §13.2 captured the rate-limit + Cloudflare findings, and
DECISIONS 2026-05-13 captured WHY the verified-wins-over-unverified
merge rule was codified. None of this would have fit cleanly into
CLAUDE.md without further bloating it.
