## What happened

**Root cause of the Hede 55 anomalies** — a mis-merge. The two anomalous
weights came from Wolfenbüttel occupation-coinage KMM records (290903
58.9 g, 291098 1.89 g «gutergros») that the absorb had stuck into the
København Speciedaler via a bare «Schou 4/5/14» collision. The current
weight-tier-1 matcher already rejects them; they were sticky pre-2026-05-22
residue. Removed from `dk-tid-97358` composed_of + phantom source URLs
surgically cleaned. KMM 290904 (PRO RELIGIONE 1627 Wolfenbüttel, 28.35 g)
stays pending the user's split verdict (nested with the real Hede 55
KMM 348808). The «thinning shows no effect» was correct behaviour — the
weights were legitimately spread across sub-variants.

**(A) Catalogue-index normalization** (`17c7e91` code, `75734e6` data).
`normalise_catalog()` (case-insensitive fold of `others: <code># N` →
typed list-field + case-insensitive value de-dup) wired into all 4
catalog-write chokepoints. The big one: a **restart-scope registry** in
`_catalog_refs` — empirically Hede 59 % / Schou 64 % / Sieg 42 % of values
collide across reigns → ruler-scoped; KM 43 % cross-entity → register;
Galster per-vol; rest global. **Sieg + Schou were bare = a §9.4 cross-reign
false-merge bug; fixed.** Rollout to all entities: 0 cross-ruler false
merges, 66 §9a museum-specimen consolidations, contained to the 3 Danish
entities.

**(B) natmus errata** (`18a5fbe`). KMM 275643 «2 Skilling 1625» wrongly
typeNumber'd «Hede 141» (= 8 Skilling 1630); corrected to Hede 134 via
`_source_errata`, corroborated by KMM 190547 (real Hede 141) + KMM 335046
(natmus's own «H. 134A» sibling) + danskmoent.

**(C) KMM citation thinning** (`7d37a92` code, `758cfba` data). 3-category
declutter: weight → untouched (§9a owns it); image-only → keep 3; neither
→ keep 1. Image presence read from cache `related.assets[type=still]`,

**Docs.** Made handoff-update at session end MANDATORY (CLAUDE.md + PB-8

## Rhinsk Gylden = wrong Fuß (later same day)

User: «Rhinsk Gylden → перенести в rhinsk_gylden_fod (Δ −1,7%, чисто). роби».
Investigation (§0b — verified, didn't trust handoff): c3h14 «1 Rhinsk Gylden»
+ c3h15 «2 Rhinsk Gylden», Christian III 1546 Flensborg, **gold** .75, were
sitting in the **silver** `8_daler_lybsk_fod`. Root cause: the Flensborg
mint-anchor rule in `auto_classify_seed_unsorted.py::_MINT_BOUND_FUSSES`
matched mint+ruler+year WITHOUT a metal check, and its `kind_by_nominal`
even deliberately mapped «Rhinsk Gylden»→kurant — so it swallowed the gold
pieces into the silver Flensborg line. §8a step-1 (metal-mismatch) was the
missing filter.

Fix in two commits:
- **`896ffef` (classifier)** — `allowed_metals` gate on the bindings +
  `_coin_matches_mint_binding` rejects metal-outside BEFORE mint/ruler/year.
  8_daler_lybsk_fod ← {silver, billon}. Systemic; future gold won't mis-route.
- **`2dc3adf` (data)** — moved c3h14/c3h15 → `rhinsk_gylden_fod` phase 0 (the
  fuss ALREADY had a Christian III Phase 0 = 1536 Roskilde galster-c3g-131, so
  the 1546 Flensburg pair extends it, not a new phase). Added fraction '2'
  (soll 6.496 rauh / 5.002 fein; c3h15 fein 4.917 → Δ −1.7 %, same as c3h14
  fraction 1 → Δ −1.7 %). New ref `danskmoent-c3-rhinsk-gylden-1546` (Galster
  130, NFM XII s.10). denmark.yml Phase-0 prose extended across all 3 surfaces
  (timeline-bar year_to 1536→1546 + title + description; fuss_periods
  hintergrund; pdate_label hiatus 27y→17y) — sourced, §7a-clean (dropped the
  single-specimen weight from the bar title). Reader-facing verification_notes
