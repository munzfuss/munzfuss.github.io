## Catalog-index sort + range-collapse (later same day)

User: «Schou-діапазони (≥3 поспіль → range), також зроби сортування всіх
індексів» with the messy example `Schou# 26, 28-29, 23-24, 25-26, 36, 9, 10`
→ wants sorted + correctly collapsed.

Implemented in `compute._compute_catalog_groups` (`94d6213`): a nested
`collapse_runs()` replaces the old per-value `sorted(set(...), key=value_key)`.
It expands every value to its integer members FIRST — so existing ranges and
overlapping/adjacent inputs merge (`23-24`+`25-26`+`26` → the int set
{23,24,25,26} → `23-26`), not a messy unsorted overlap. Runs of ≥3 consecutive
ints collapse to `min-max`; 1-2 runs stay individual numbers; non-integer
tokens (letter sub-variants `93A`, dotted `77.1`, register-qualified) never
collapse and keep the existing `value_key` natural order (parent before
sub-variant). Applies to ALL index groups, not just Schou.

User example → `9, 10, 23-26, 28, 29, 36`. Verified on KM-42 (dk-tid-163034):
`Schou# 21, 24, 25, 28, 32-36, 40, 42, 46-48, 51, 52, 56-59, 61, 62, 64, 68,
71-76, 83, 89, 90, 93, 95, 97-99` (was a 38-number flat unsorted-ish list).
2-element runs render as two numbers per the stated ≥3 rule — one-line
threshold flip if ≥2 is wanted later.

## Davenport volume-fold (later same day)

User flagged `Dav# 3668, 3668D, EC II 3668, EC II 3668D` / `Dav# 3523, EC II#
3523` — bare + volume-qualified duplicates; asked to google «EC II» and decide
whether the bare form is droppable. Researched (Internet Archive + Google Books):
**EC = Davenport «European Crowns», century-tagged volumes — EC I (1484-1600),
EC II (1600-1700, Galesburg 1974), EC III (1700-1800).** Numbering is CONTINUOUS,
so «Dav 3668» = «EC II 3668» (same ref, volume-implicit vs explicit). Strongest
proof: the project's OWN data has Numista citing one coin (royal_holstein 1
Ducat) both ways — a side-effect of the multi-KM-session's dav accumulation.
Conclusion: bare form is droppable; keep the volume-qualified.

Implemented (`bc1f9d7`): step 4 in `catalog_codes.normalise_catalog` — when the
dav list holds both a bare «N» and a volume-qualified «<VOL> N» (same trailing
number; VOL ∈ EC/GT/SG/BrSL/…), drop the bare. A bare with NO matching qualified
is kept. Runs on every merge + absorb-blanket. Materialized on the existing 145
coins / 9 entities via a hygiene-only text pass (`92557b3`): 148 bare lines
dropped, **pure deletion** (0 added), 0 merge re-convergence, 0 residual bare-vs-
EC dups in the render. validate OK.
